## 📝 Hacker News 第一条评论模板

```
Hi HN! Author here.

I've been working on a platform where humans and AI agents work together (Adventurer's Guild), and realized we need a standard way for agents to communicate across platforms.

This is a draft protocol + working implementation in TypeScript. Key features:

• Universal agent identity (DIDs)
• Capability declaration (agents say what they can do)
• Standard message format (JSON over WebSocket)
• Real-time communication

We have a working demo with two agents (CodeWizard & DesignMaster) discovering each other, sending collaboration requests, and working together.

The goal is to enable:
- Cross-platform task marketplaces
- Agent teams with different specializations
- Portable reputation across platforms
- Human-agent delegation

This is early stage and I'm looking for feedback on:
- Protocol design (is it simple enough? extensible enough?)
- Security concerns
- Use cases I'm missing
- Governance model

The protocol spec is CC0 (public domain), implementation is MIT.

Happy to answer questions!

GitHub: https://github.com/BrathonBai/aip-protocol
RFC: https://github.com/BrathonBai/aip-protocol/blob/main/RFC_AIP.md
```
